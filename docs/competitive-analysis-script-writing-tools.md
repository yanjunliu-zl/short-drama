# 剧本创作产品竞品全景分析（2026 Q3）

> 聚焦剧本创作环节 — 短剧 Agent 内置编剧模块、通用 LLM 剧本创作、专业编剧工具、海外 AI 写作产品。分析维度：创作能力、格式规范、长文本连贯性、角色一致性、商业化成熟度。

---

## 一、为什么单独分析剧本创作

上一份竞品分析覆盖了"生产工具层"（11 家 AI 成片 Agent），但那些产品的核心价值在「剧本→视频」。**剧本创作是价值链最上游**——下游所有工具的输入质量上限由剧本决定。

**核心数据**：
- 全行业 AI 短剧爆款率：0.12%-0.16%，破亿作品不足 150 部
- 单月供给 1.46 万部，供给过剩但精品极度稀缺
- AI 生成剧本普遍"看着唬人，实则不能用"
- 头部短剧公司反馈：**AI 剧本缺乏人物弧光和情感层次，需要大量人工改写**

**我们的位置**：剧本生成层 → 连接"创意想法"和"可拍剧本"的桥梁。如果这层做不好，下游所有工具都在浪费时间。

## 三、第一类：短剧 Agent 内置编剧模块

这些产品的核心能力在视频生成，编剧模块是入口而非核心。

### 3.1 小云雀 AI 编剧（字节跳动）

| 维度 | 评估 |
|------|------|
| **输入方式** | 支持上传 10 万字小说/IP 文本；支持"一句话生成剧本" |
| **剧本格式** | 内部格式，自动分集 + 分镜标记 |
| **角色管理** | "全局角色管理"——声称支持跨集一致性，实测角色外貌偏差率 >35% |
| **长文本能力** | 10 万字输入上限，但超过 5 万字质量明显下降 |
| **短剧适配** | ★★★★☆ — 深度绑定红果短剧，理解付费点和钩子节奏 |
| **输出可用率** | 首稿可用率约 30%-40%，需要大量人工修改 |
| **定价** | 39 元/月（包含在成片积分内，不单独收费） |

**优势**：唯一全闭环——剧本→成片→分发→变现在一个体系内。数据飞轮可反哺模型（红果数据 → 训练 → 更好的剧本理解）。

**核心短板**：
1. 编剧模块不是独立产品，只是成片管线的入口
2. 角色一致性问题至今未解决（>35% 偏差率）
3. 生成剧本同质化严重——"重生+复仇+甜宠"三板斧
4. 缺乏深度人物塑造能力和情感弧光设计

**判断**：小云雀的编剧能力是"够用但不出彩"。它的核心价值在 Seedance 生成视频，不在生成剧本。

---

### 3.2 巨日禄 AI 编剧

| 维度 | 评估 |
|------|------|
| **输入方式** | 支持 5 万字长文本解析；AI 剧本生成 + 智能分镜一体化 |
| **剧本格式** | 自动生成分集 + 场景 + 分镜表 |
| **角色管理** | 多主体融合技术，跨场景一致性 ≥95%，角色三视图资产化，误差率 <3% |
| **长文本能力** | 5 万字解析能力，适合网文漫改 |
| **短剧适配** | ★★★☆☆ — 更偏二次元漫剧，写实短剧表现一般 |
| **输出可用率** | 漫剧可用率较高（≥70%），真人短剧剧本可用率低 |
| **定价** | 按剧集包年/充值积分制 |

**优势**：角色一致性行业最佳（<3% 误差率 vs 小云雀的 >35%）。多主体融合技术值得深入研究。

**核心短板**：
1. 专注漫剧赛道，写实短剧剧本生成能力弱
2. 需要用户深度参与（"手搓式"），不是"一键出剧本"
3. 缺乏人类编剧的"灵气"——生成的剧本偏公式化

## 四、第二类：通用 LLM 用于剧本创作

这是我们的主要竞争对手——编剧们实际用这些工具写剧本。

### 4.1 Claude（Anthropic）

| 维度 | 评估 |
|------|------|
| **核心优势** | 200K token 上下文窗口（可一次性读完整部剧本）；**业界公认的对话/剧本写作最优** |
| **对话质量** | ★★★★★ — 最自然的角色对话，能区分不同角色的语言风格和情感层次 |
| **长文本连贯性** | ★★★★★ — 200K 上下文窗口，60 集剧本结构可在一次对话中保持 |
| **创意丰富度** | ★★★★☆ — 比 ChatGPT 更有"文采"，句式多样性更好 |
| **中文能力** | ★★★☆☆ — 英文创作顶级的，中文有提升但不如 DeepSeek |
| **格式支持** | ★★☆☆☆ — 无剧本格式，纯对话界面，需手动处理格式 |
| **角色记忆** | ★★★☆☆ — 会话内记忆强，但跨会话不持久 |
| **定价** | Free / Pro $20/月 / Max $100-200/月 |

**剧本创作适用场景**：英文本土化剧本（our P2）、复杂角色对话、长剧本结构分析、剧本润色。

**对我们的威胁**：如果用户直接用 Claude 写剧本，为什么还需要我们的平台？

| 维度 | 评估 |
|------|------|
| **核心优势** | 最通用的创作工具；Canvas 模式支持文档内编辑；Custom GPT 生态 |
| **对话质量** | ★★★☆☆ — 偏"安全"和"解释性"，缺乏角色辨识度 |
| **长文本连贯性** | ★★★☆☆ — 约 30 页后开始丢失角色细节 |
| **创意丰富度** | ★★★☆☆ — 需要大量 prompt 引导才能产出有创意的内容 |
| **中文能力** | ★★★☆☆ — 中文创作能力一般，不如 DeepSeek |
| **格式支持** | ★★☆☆☆ — 无剧本格式 |
| **定价** | Free / Plus $20/月 / Pro $200/月 |

**剧本创作适用场景**：多类型脚本（YouTube/广告/播客）、头脑风暴、大纲规划。

| **格式支持** | ★☆☆☆☆ — 纯文本，无任何格式辅助 |

---

### 4.4 通用 LLM 的共性短板（剧本创作视角）

| 短板 | 描述 | 影响 |
|------|------|------|
| **无短剧格式** | 生成的文本没有"第N集""【场景】""角色名："等短剧行业标准格式 | 需要人工格式化 |
| **无角色系统** | 角色信息靠 prompt 维护，长对话中逐渐漂移 | 30 集以上角色走形 |
| **无情节结构** | 不会主动设计 cliffhanger、付费点、三幕结构 | 生成的"剧本"像小说而非短剧 |
| **无行业合规** | 不理解广电备案要求、平台内容规范 | 生成内容可能违规 |
| **无数据闭环** | 不能从"用户接受率""重写率"中学习优化 | 每次生成质量相同 |
| **无管线衔接** | 生成的文本不能直接导入 AI 成片工具 | 需要人工搬运 |

**而这些恰好是我们能解决的**——不是跟 Claude 比写作，而是在剧本→成片管线中提供无可替代的价值。

## 七、改进建议

### P0 — 立即做（可能决定产品生死）

#### 7.1 情节结构模板系统

**对标**：Scrite 的 Save the Cat beat sheet + 创一 AI 的结构化流程。

```
短剧需要的是结构化的"爽点模板"，而非自由创作：

模板示例 — 重生复仇 24 集:
  第1集:  重生场景 + 前世死亡闪回 → 钩子: "这一次我要让所有人付出代价"
  第2集:  初次展示前世记忆的优势 → 钩子: 遇到前世的关键人物
  第3集:  第一次小胜利 → 钩子: 发现更大的阴谋线索
  第4-5集: 建立盟友 + 初步积累力量 → 每集有一个小反转
  第6集:  遭遇第一个阻碍 → 钩子: 看似失败实则布局
  ...
  第12集: 中点大反转 → 钩子: "原来他/她也重生了"
  ...
  第24集: 阶段性复仇成功 + 感情线进展 → 钩子: 更大的敌人浮出水面

每种短剧类型（重生、霸总、虐恋、悬疑）都应有对应模板。
```

**实现**：
- 新增 `plot_templates.py` — 10+ 种短剧类型的集级结构模板
- 剧本生成时选择模板，AI 按模板节奏填充内容
- 模板包含：每集目标、反转点位置、钩子类型、对话/动作比例

#### 7.2 角色一致性引擎 v2

**对标**：巨日禄的多主体融合技术（<3% 误差）+ Sudowrite Story Bible。

当前 V2 pipeline 的角色提取是每章独立的——跨章可能因为 LLM 的随机性导致同一角色描述不同。

```
改进方案:
  1. 全局角色档案 (已有 AssetLibrary，需深度集成到 pipeline)
  2. 角色特征锁定: 在生成每章剧本时，显式注入角色档案作为约束
     - 外貌锁: "顾清寒：女，15岁，白色长袍，左眼角有泪痣"
     - 性格锁: "顾清寒：性格清冷但重情，表面柔弱实则步步为营"
     - 对白锁: "顾清寒对白风格：简洁、克制、偶尔冷幽默"
  3. 跨集角色一致性校验: 每集生成后用 QualityJudge 的角色一致性维度评分
```

#### 7.3 多版本生成 + A/B 对比

**对标**：Sudowrite 的 "Brainstorm" 替代方案 + 我们的 QualityJudge A/B 对比。

```
流程:
  用户输入需求
    → 同时生成 3 个剧本版本（不同叙事角度/节奏风格）
    → QualityJudge 自动评分 + 对比
    → 用户选择最佳版本
    → 或在最佳版本基础上融合其他版本优点
```

QualityJudge 的 A/B 对比已有基础，只需加多版本生成逻辑。

### P1 — 近期做（提升产品壁垒）

#### 7.4 用户偏好学习

当前 `user_preferences.py` 有基础框架，但未深度集成。

```
改进:
  - 记录用户选择的剧本版本、修改行为
  - 学习用户的风格偏好（偏甜宠还是偏虐？偏快节奏还是情感细腻？）
  - 下次生成时自动倾向于用户偏好的风格
  - 构建用户画像: 题材偏好、长度偏好、角色类型偏好
```

#### 7.5 钩子/付费点自动设计

短剧的核心商业逻辑是"让用户点开下一集"和"让用户付费"。AI 需要理解这一层。

```
实现:
  - 在情节模板中标注付费点位置（通常第10-12集、第18-20集）
  - AI 在这些位置自动生成更强的钩子和信息差
  - 钩子质量用 QualityJudge 的 cliffhanger_quality 维度评分
  - 不达标自动重写
```

#### 7.6 团队协作

**对标**：LibTV 的实时协作 + 统一资产库。

```
实现:
  - 剧本项目共享（多编剧协作同一剧本）
  - 角色资产/场景模板团队共享
  - 评论/批注系统
```

### P2 — 中长期（构建生态壁垒）

#### 7.7 剧本→数据反馈闭环

```
用户使用剧本 → 生成成片 → 平台数据反馈（播放量、完播率、付费率）
  → 好的剧本特征自动提取 → 反馈到 prompt 模板和情节结构
  → 后续生成的剧本越来越符合"爆款规律"
```

这是小云雀的核心壁垒（红果数据飞轮），我们目前做不到，但可以先设计数据接口。

#### 7.8 行业格式标准

推动短剧剧本的标准化格式，成为行业标准制定者：
- 提案：Short Drama Script Format (SDSF)
- 包括：集结构 + 场景标记 + 对白格式 + 钩子标注 + 付费点标注 + 角色出场表
- 所有下游工具（小云雀/LibTV/巨日禄）都能解析这个格式

## 九、核心判断

**通用 LLM（Claude/ChatGPT/DeepSeek）不是我们的竞争对手——它们是我们的原材料。** 我们的价值不在"写得好"，而在：

1. **格式壁垒**：生成的剧本直接符合短剧行业标准，可直接导入下游工具
2. **行业知识**：理解 hook/cliffhanger/paywall/节奏/合规——通用 LLM 不理解这些
3. **管线集成**：剧本→分镜→图像→视频→分发，一站式
4. **数据反馈**：剧本被使用→被拍出来→被用户观看→数据回流优化——通用 LLM 没有这个闭环

**但有一个硬性条件：如果我们的剧本质量不能持续超越"用户直接找 DeepSeek/Claude 写"的水平，所有管线优势都是空中楼阁。** 情节结构模板系统是当前最紧迫的改进项——它能让剧本从"AI 随机发挥"变成"按行业验证过的结构创作"。
