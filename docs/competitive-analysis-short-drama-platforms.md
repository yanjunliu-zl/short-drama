# 短剧生产与分发平台竞品全景分析（2026 Q3）

> 覆盖 AI 生产工具 11 家、国内分发平台 8 家、海外出海平台 12 家。分析维度：技术能力、产品体验、商业模式、生态壁垒、团队适配。

---

## 一、执行摘要

2026 年短剧行业进入"AI 工业化"拐点。上半年国内 AI 漫剧单月上线超 14,000 部，红果短剧 DAU 突破 1 亿。供给端从"手工作坊"切换到"AI 流水线"，20 余家 AI 短剧 Agent 在 18 个月内涌入市场。

**核心判断**：
- **生产工具层**：已分化为"大厂自研""创业垂直深耕""模型聚合"三大阵营，目前没有绝对赢家
- **国内分发层**：红果一家独大（DAU 过亿），腾讯（火龙漫剧）、优酷（漫剧频道）正在追赶，分账规则成为抢团队的关键变量
- **海外分发层**：DramaBox（月活 3106 万）和 ReelShort 双强格局，但 80%-90% 项目无法回本，投流成本吞噬利润
- **关键机会窗口**：AI 降低制作门槛后，"好故事"成为终极稀缺资源。剧本质量直接决定一部剧是 0.16% 的爆款还是剩余 99.84% 的炮灰

**对我们的启示**：我们处于"生产工具层"上游——剧本生成环节。下游工具（小云雀、LibTV、巨日禄）解决的是"剧本→成片"问题，我们解决的是"创意→剧本"问题。两者是互补关系，不是竞争关系。

## 三、AI 生产工具层——核心竞品深度剖析

这是与我们最相关的层级。以下 11 家按阵营分析。

### 3.1 大厂自研阵营

#### 3.1.1 小云雀 AI（字节跳动）

| 维度 | 详情 |
|------|------|
| **上线时间** | 2026 年 3 月 19 日短剧 Agent 功能上线 |
| **底层模型** | Seedance 2.0（被行业评价为"当前最强视频生成模型"） |
| **产品矩阵** | 小云雀（专业 Agent）+ 即梦（轻量创作）+ 剪映（编辑）+ 豆包（对话） |
| **用户规模** | App DAU 超 80 万（环比 +54%），下载量环比 +122% |
| **定价** | 39 元/月（1200 积分），Seedance 2.0 Fast 模式 11 积分/秒 |
| **产能** | 单线程生成，高峰期排队最长 8 小时 |

**核心技术能力**：
- 支持上传 10 万字剧本，一键直出视频成片
- 具备"故事理解"与"全局角色管理"能力
- 多镜头叙事自动规划、音画一体同步生成
- 被评价为具备"导演思维"——能理解什么样的开头留住用户、什么节奏撑到付费点

**生态闭环（这是小云雀最深的护城河）**：
```
Seedance 模型 → 小云雀/即梦/剪映 工具 → 抖音/红果 分发 → 穿山甲 变现
                    ↑                                          │
                    └──────── 用户行为数据回流 ─────────────────┘
```

- 独享抖音、番茄小说海量爆款短剧与用户偏好数据训练模型
- 番茄小说 60 万签约作者 + 38 万册电子出版物提供 IP 蓄水池
- "番茄火什么，红果就拍什么"——IP 到成片的转化链路极短

**核心短板**：
1. **角色一致性差**：无全局角色资产库，跨分集人物五官/服饰偏差概率超 35%——这是多集短剧的致命伤
2. **单线程瓶颈**：不支持并行生成，高峰排队数小时，产能受限
3. **成本偏高**：60 集短剧综合算力成本约 3000 元（单集约 56 元），对比巨日禄 200 元/分钟有明显劣势
4. **复杂叙事崩坏**：多人对话镜头、复杂动作场面易崩坏
5. **内部同质化**：豆包、即梦、剪映、小云雀底层模型相同，仅开放功能和额度不同——产品矩阵有"套壳"嫌疑

**战略判断**：小云雀不靠自身赚钱——字节的策略是"免费换供给"，用工具吸引创作者，内容回流到抖音/红果分发体系变现。这是只有字节玩得起的打法。

#### 3.1.3 万镜一刻（阿里云）

| 维度 | 详情 |
|------|------|
| **上线时间** | 2026 年 5 月 21 日阿里云峰会发布 |
| **底层模型** | 万相 Wan 2.7 + HappyHorse + Qwen-image + Z-image |
| **核心差异** | "主体创作"引擎——角色/场景/道具先建多视角主图存入视觉档案 |
| **产品形态** | 三种工作流：故事板、无限画布、多 Agent 协同对话式成片 |
| **商业化** | 工具平台 + API + 品牌套件三条线 |

**技术亮点**：
- **"主体创作"是真正的差异化**：跨镜头不换脸、风格不漂移，素材可复用为资产——这正好解决了小云雀的最大痛点（角色一致性 35% 偏差）
- 五模块完整：剧本智能解析 → 故事板生成 → 主体创作 → 在线剪辑 → 资产管理

**战略短板**：
- 阿里没有内容消费场（没有抖音/红果这样的分发平台），创作者用万镜一刻做完内容后需要自行找分发渠道
- 底层偏向服务电商团队（阿里电商基因），影视创作场景的适配深度待验证
- "除了字节，其他大厂做影视 Agent 都是吃力不讨好"——行业普遍判断

### 3.2 创业垂直深耕阵营

#### 3.2.1 LibTV（哩布哩布 AI）⭐⭐⭐⭐⭐

| 维度 | 详情 |
|------|------|
| **上线时间** | 2026 年 3 月 18 日 |
| **定位** | 一站式专业 AI 短剧工作台 |
| **核心设计** | "无限画布 + 节点式工作流"，像搭积木一样调灯光、调机位 |
| **模型集成** | 可灵 3.0、Seedance 2.0、HappyHorse 1.0、Wan 2.6、Seedream 5.0、Midjourney V7、Mureka 等 |
| **商业化** | 团队版 2026 年 5 月上线，超 300 家短剧公司/影视工作室成为 B 端客户 |
| **收入表现** | 上线首月单日收入破百万美元，5 月收入达首月 13 倍，母公司估值直逼 20 亿美元 |

**为什么 LibTV 值得重点研究**：

**1. 多模型聚合策略——不绑定单一模型**
- 与字节（绑定 Seedance）、快手（绑定 Kling）不同，LibTV 集成了几乎所有主流视频/图像/音频模型
- 创作者可以在同一画布上对不同镜头调用不同模型——"文戏用 Seedance，打斗用可灵"
- 某模型升级或降价，创作者可立即切换，无迁移成本

**2. 专业工作流设计——面向"创作者"而非"消费者"**
- 角色三视图、多机位九宫格、剧情推演四宫格、3D 导演台
- 节点式工作流允许精细控制灯光、机位、角色位置
- 面向 3-20 人专业工作室，而非个人用户

**3. Agent 原生——为 AI Agent 协作设计**
- Agent 可通过 Skill 接口直接调用 LibTV 工作流
- 这意味着我们的剧本生成 Agent 可以直接对接 LibTV 的制作管线
- 被评价为"综合实力领先"的短剧制作平台

**4. 协同与资产沉淀——构建团队级护城河**
- 多人实时协作共享画布，统一团队资产库
- 角色资产、场景模板、分镜预设可沉淀复用
- 团队使用越久，迁移成本越高——这是 SaaS 经典的"数据引力"护城河

**短板**：
- 面向"技术极客"和专业创作者，新手学习曲线较陡
- 年费 2888 元/团队，个人创作者有更便宜的选择
- 无自有分发渠道——只做工具，不做平台

#### 3.2.2 巨日禄 AI ⭐⭐⭐⭐

| 维度 | 详情 |
|------|------|
| **定位** | 工业级零基础漫剧制作平台，专注二次元/动漫短剧批量量产 |
| **目标用户** | 2-5 人小型团队、批量产出的 MCN 工作室 |
| **核心能力** | 多主体融合技术，跨场景一致性 ≥95%，角色三视图资产化引擎误差率 <3% |

**差异化优势**：
1. **角色一致性行业最佳**：≥95% 跨场景一致率，误差率 <3% —— 这是小云雀（偏差 35%）的 10 倍以上
2. **极致成本**：200 元/分钟，对比小云雀 3000 元/60 集有明显优势
3. **批量产能**：单人单日可产出约 30 集，自动审/自动抽/自动剪——后期几乎无需人工干预
4. **一键转绘**：2D/3D/AI 真人风格互转，支持英/日/韩/西/法/泰多语种本地化重拍——这是出海利器
5. **多主体融合技术**：近百种风格（国漫/日漫/韩漫）

**短板**：
- 二次元/漫剧赛道专精——写实/科幻大场景表现力偏弱
- 操作偏"手搓式"，新手需一定经验积累
- 充值门槛较高（按剧集包年/充值积分制）
- 品牌和声量远不如小云雀，获客依赖口碑传播

#### 3.2.3 有戏 AI（风平智能）

| 维度 | 详情 |
|------|------|
| **定位** | 全流程一站式 AI 短剧工具，兼顾新手与创作效率 |
| **成本** | 69 元/月商用，视频生成 0.1 元/秒（行业最低） |

**差异化优势**：
1. **极致低价**：视频生成 0.1 元/秒，算力成本比行业均值低 30%-50%
2. **新手友好**：线性流程清晰（剧本→主体→分镜→故事板→成片），1 小时内可完成单集成片

6. **导出剪映工程**：与剪映生态衔接

**短板**：
- 角色一致率 90%-95%，略逊于巨日禄
- 品牌知名度低于小云雀/LibTV
- 由数字人公司转型，影视工业积累不如爱奇艺/阿里深厚

## 六、战略洞察与行动建议

### 6.1 行业格局判断

```
                    AI 能力 (强)
                         │
                    ┌────┼────┐
                    │ 小云雀  │  ← 模型强，但角色一致性短板明显
                    │ 万镜一刻│  ← 主体创作引擎是亮点，但缺分发
                    └────┼────┘
                         │
    ─────────────────────┼─────────────────────
    分发弱              │              分发强
    (需自寻渠道)        │          (平台自带流量)
    ┌────┼────┐    ┌────┼────┐
    │ LibTV   │    │ 可灵AI │  ← 快手生态
    │ 巨日禄  │    │ 纳逗Pro│  ← 爱奇艺生态
    │ 有戏AI  │    └────┼────┘
    └────┼────┘         │
         │              │
         ▼              ▼
    "卖铲子"型      "管道"型
    工具变现        生态变现
```

**关键判断**：
- **字节是唯一全闭环玩家**，短期内不可复制
- **LibTV 是工具层最强的独立玩家**，多模型聚合 + Agent 接口与我们高度互补
- **巨日禄的角色一致性技术是所有平台的标杆**，从技术路线上值得深入研究
- **除了字节，其他大厂做影视 Agent 都是"吃力不讨好"**——没有内容消费场的工具，难以形成商业闭环

### 6.2 我们的定位与机会

```
我们的位置：剧本生成层

    ┌──────────────────────────────────────────┐
    │          我们：剧本智能生成引擎           │
    │                                          │
    │  输入：主题、风格、长度、梗概、人物设定    │
    │  输出：完整多集剧本（含场景、对白、钩子）  │
    │  核心价值：好故事 → 爆款概率提升          │
    └──────────────────────────────────────────┘
                        │
                        │ 剧本输出
                        ▼
    ┌──────────────────────────────────────────┐
    │  下游工具（小云雀/LibTV/巨日禄/有戏AI）  │
    │                                          │
    │  输入：完整剧本                           │
    │  输出：成片视频                           │
    │  核心价值：剧本→视频的自动化转换          │
    └──────────────────────────────────────────┘
```

**我们的结构性优势**：
1. **不与任何下游工具竞争**——我们是上游供应商，所有 AI 成片工具都是潜在合作伙伴
2. **"好故事"是全行业终极瓶颈**——当下游工具让"拍出来"不再难，"拍什么"成为唯一的差异化
3. **全行业爆款率 0.16%**——剧本质量是决定这 0.16% 的最核心变量
4. **监管趋严利好头部**——2026 年 4 月起 AI 短剧需"先备案后上线"，广电总局划定投资红线，低质内容加速出清

### 6.3 行动建议

| 优先级 | 行动 | 说明 |
|--------|------|------|
| **P0** | 剧本质量对标人类编剧 S 级水准 | 0.16% 的爆款率意味着：只有最好的剧本值得被拍出来 |
| **P0** | 输出格式兼容下游工具 | 确保我们的剧本可以直接导入小云雀（10 万字支持）、LibTV（脚本工作流）、巨日禄（5 万字解析） |
| **P1** | 对接 LibTV Agent Skill 接口 | LibTV 是唯一原生支持 Agent 调用的工具，可以形成"剧本 Agent → 成片 Agent"的自动化管线 |
| **P1** | 建立剧本质量自动评估体系 | 用 AI Judge + 人类编辑 double-check，在剧本环节就筛掉 90%+ 的低质内容 |
| **P2** | 探索剧本→成片的端到端自动化 | 配合 LibTV Skill 接口或直接调用 Seedance API，实现"输入梗概→输出成片" |
| **P2** | 关注海外本土化剧本需求 | 海外本土剧占比从 21% 升至 44%，多语言剧本生成是蓝海 |

### 6.4 风险提示

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 字节全线封闭生态 | 中 | 高 | 保持多模型/多工具兼容，不绑定单一平台 |
| 下游工具自研剧本能力 | 中 | 中 | 小云雀已支持 AI 剧本生成——但我们做的是"专业级"，不是"凑合用" |
| 广电监管进一步收紧 | 高 | 中 | 剧本合规检查能力是护城河，不是成本 |
| 行业爆发期结束，进入淘汰 | 高 | 低 | 我们是工具层，淘汰期反而利好——大家更需要高质量工具降本增效 |
| 开源模型追赶闭源 | 中 | 低 | 同上——更廉价的视频生成会进一步放大"好剧本"的稀缺性 |
